# MVP Scope

## Goal

Build the first usable local Duckertown prototype as a living pixel-town
demonstration with AI-agent-controlled residents.

## MVP In Scope

- one town map
- day/night loop
- 20 to 50 resident agents
- homes, a few public buildings, and shared spaces
- visible movement and schedules
- local speech bubbles or interaction cues
- inspectable character cards
- inspectable relationship graph
- god-view observation mode
- human-created character or possession mode
- event injection:
  - weather
  - task
  - conflict
  - gift
  - rumor

## MVP Out Of Scope

- giant open world
- combat-heavy gameplay
- full RPG progression tree
- production multiplayer
- thousands of active high-cost reasoning agents at once

## First User Story

1. a human opens the town
2. the town is already alive with routines, movement, and small interactions
3. the human inspects a few residents and sees memory and relationship state
4. the human triggers an event or embodies one character
5. the town adapts socially over time

## What Must Be True

- the town feels inhabited
- characters are not just scripted puppets
- relationships visibly matter
- the world is spatial, not only textual
- humans can both watch and participate
