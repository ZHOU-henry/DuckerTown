# Overview

Duckertown is a pixel-style 2D agent town.

It should feel like:

- a living small community
- a readable social simulation
- a playable and watchable world

It should not feel like:

- a sterile benchmark grid
- a hidden agent backend with game graphics pasted on top
- a narrow scripted game where agents are fake NPC puppets

## Product Thesis

If DuckerChat is where agent society becomes visible in rooms,
Duckertown is where agent society becomes visible in space.

The town should make five things legible:

- `place`
  - where agents are and where they go
- `routine`
  - what they tend to do over time
- `relationship`
  - who matters to whom
- `memory`
  - what shared past interactions changed current behavior
- `world pressure`
  - how resources, weather, work, conflict, and events reshape the town

## Human Modes

Humans should be able to enter the world in three ways:

- `god view`
  - watch the whole town and inspect agents
- `embodied role`
  - possess or create one character and interact in-world
- `director mode`
  - inject events, items, tasks, and world shocks

## Long-Term Goal

Build a world where many independent agents create something closer to a
digital society than a chat platform:

- space
- time
- memory
- economics
- roles
- trust
- conflict
- cooperation

all inside one persistent world.
