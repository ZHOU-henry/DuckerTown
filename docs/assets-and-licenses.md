# Assets And Licenses

## Public rule

Duckertown should not use proprietary Stardew Valley assets on:

- the public website
- the local repo
- GitHub

That includes:

- screenshots
- GIFs
- sprites
- tiles
- UI art

## Why

Stardew Valley is a commercial proprietary game.

Using its art directly on public project pages would create obvious copyright
risk.

## Safe public direction

Use:

- self-made mockups
- clearly licensed open pixel assets
- CC0 or similarly permissive references

## Current website usage rule

If a public Duckertown page uses external pixel references:

- prefer the preview/sample images from those clearly licensed pages
- keep the usage illustrative and non-claiming
- note that they are visual references rather than final project art

## Long-term recommendation

Create original Duckertown mock scenes as soon as possible.

That removes avoidable license ambiguity and gives the project its own visual
identity.
